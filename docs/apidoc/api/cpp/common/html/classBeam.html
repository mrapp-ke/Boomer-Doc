Searches for the best refinements of the rules that are kept track of by a given beam and updates the beam accordingly.

Parameters
ruleCompareFunctionAn object of type RuleCompareFunction that defines the function that should be used for comparing the quality of different rules
beamPtrA reference to an unique pointer of type Beam that represents the beam to be updated
beamWidthThe number of rules the new beam should keep track of
featureSamplingA reference to an object of type IFeatureSampling that should be used for sampling the features that may be used by potential refinements
keepHeadsTrue, if further refinements should predict for the same labels as before, false otherwise
minCoverageThe number of training examples that must be covered by potential refinements
numThreadsThe number of CPU threads to be used to search for potential refinements of a rule in parallel
rngA reference to an object of type RNG that implements the random number generator to be used
Returns
True, if any refinements have been found, false otherwise

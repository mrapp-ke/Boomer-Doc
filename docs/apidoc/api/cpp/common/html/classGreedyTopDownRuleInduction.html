Detailed Description

An implementation of the type IRuleInduction that allows to induce individual rules by using a greedy top-down search.

minCoverageThe minimum number of training examples that must be covered by a rule. Must be at least 1
maxConditionsThe maximum number of conditions to be included in a rule's body. Must be at least 1 or 0, if the number of conditions should not be restricted
maxHeadRefinementsThe maximum number of times, the head of a rule may be refinement after a new condition has been added to its body. Must be at least 1 or 0, if the number of refinements should not be restricted
recalculatePredictionsTrue, if the predictions of rules should be recalculated on all training examples, if some of the examples have zero weights, false otherwise
numThreadsThe number of CPU threads to be used to search for potential refinements of a rule in parallel. Must be at least 1
